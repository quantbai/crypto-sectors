## `classification/sector.csv`

232 rows, one per coin. Committed source of truth; edit this file to change any classification.

| Column | dtype (parquet) | Nullable | Notes |
|---|---|---|---|
| `symbol` | `string` | No | Uppercase trading symbol (e.g. `BTC`). |
| `sector` | `string` | No | Flat sector label. Must be one of the 15 valid values listed below. |
| `role` | `string` | No | `FACTOR` or `RESIDUAL`. Governs group-neutralization behaviour. |
| `cg_id` | `string` | No | CoinGecko slug (e.g. `bitcoin`). Primary identity anchor across rebrands. |
| `okx_instid` | `string` | No | OKX USDT-margined perpetual instrument ID (e.g. `BTC-USDT-SWAP`). |
| `audit_from` | `date` | No | Date from which this classification was applied. |
| `prev_label` | `string` | Yes | Previous sector label before last reclassification. `<NA>` if no prior label. |
| `note` | `string` | Yes | Free-text rationale for classification or reclassification. May be `<NA>`. |

`classification/sector.parquet` is a byte-for-byte parquet mirror of the above with identical column names and dtypes. It is regenerated by `python scripts/build_sector_field.py` and must not be edited directly.

## `classification/sector_panel.parquet`

Point-in-time (PIT) label matrix consumed directly by the `group_neut` kernel.

| Dimension | dtype | Notes |
|---|---|---|
| Index (rows) | `datetime64[ns]` | One row per calendar day, 2019-11-27 â†’ 2026-06-13. |
| Columns | `string` | One column per symbol (232 total). |
| Cell values | `StringDtype` (pandas) | The sector label string on that date, or `<NA>`. |

**`<NA>` semantics.** A cell is `<NA>` for exactly two reasons:

1. **Pre-listing**: `date < audit_from` for that coin â€” the coin was not yet classified/listed.
2. **No returns data**: the coin is in `sector.csv` but `data/returns.parquet` has no observation for that date.

Neither reason indicates a classification error. Downstream consumers must `.dropna()` before `group_neut` operations. The production kernel additionally applies a per-day min-group collapse: any FACTOR group with fewer than 3 live members on a given day is pooled into OTHER before demeaning (see FACTOR/RESIDUAL contract below).

## `classification/sector_roles.json`

Maps each of the 15 sector labels to its metadata object.

```
{
  "<label>": {
    "role":             string,   // "FACTOR" or "RESIDUAL"
    "n":                integer,  // number of coins assigned this label
    "mean_resid_corr":  float,    // mean intra-group market-residual pairwise correlation
    "cohesion_p":       float     // permutation p-value (10k-perm null)
  },
  ...
}
```

All 15 labels are present. `mean_resid_corr` and `cohesion_p` are from the production OTHER-collapse validation kernel (10,000 permutations).

## FACTOR / RESIDUAL contract

Every label carries a `role` of either `FACTOR` or `RESIDUAL`. This governs the `group_neut` production kernel.

**FACTOR** labels have mean intra-group market-residual pairwise correlation that is materially above the permutation null â€” a real shared return factor exists and demeaning within the group removes it. The discriminator is **effect size** (mean intra-group market-residual correlation vs null mean), not p-value. A large group can be p-significant with near-zero effect size (e.g. `defi` has cohesion p=0.0001 but mean resid corr +0.003, which is ~indistinguishable from the null in magnitude â€” it is RESIDUAL). See `validation/sector_okx/REPORT.md` Â§1.3.

| Label | Mean resid corr | Role |
|---|---|---|
| `privacy` | +0.55 | FACTOR |
| `captive_franchise` | +0.30 | FACTOR |
| `value_transfer` | +0.23 | FACTOR |
| `compute_storage` | +0.18 | FACTOR |
| `meme` | +0.10 | FACTOR |
| `eth_scaling` | +0.04 | FACTOR |
| `metaverse_gaming` | +0.04 | FACTOR |
| `interoperability` | +0.03 | FACTOR |
| `smart_contract_platform` | +0.013 | FACTOR (marginal â€” large-n effect) |
| `information_technology` | +0.011 | FACTOR (marginal â€” large-n effect) |

**RESIDUAL** labels have intra-group correlation indistinguishable from the permutation null. Demeaning within a RESIDUAL group is approximately a no-op. In `group_neut` these coins are pooled together with any sub-3-member FACTOR groups into a single OTHER pool, then market-neutralized as a block.

| Label | Mean resid corr | Role |
|---|---|---|
| `defi` | +0.003 | RESIDUAL |
| `ai_infrastructure` | -0.002 | RESIDUAL |
| `oracles_data` | +0.008 | RESIDUAL |
| `media_content` | +0.005 | RESIDUAL |
| `OTHER` | â€” | RESIDUAL (catch-all) |

**Production kernel rule**: on each day, demean returns within each FACTOR label that has â‰Ą 3 live members; pool all RESIDUAL-labelled coins and any FACTOR group with < 3 live members into OTHER, then demean that OTHER pool. This conservative kernel is what the published validation VR numbers measure.

**Honesty notes** (do not soften):

- `ai_infrastructure` and `oracles_data` are RESIDUAL organizational labels retained for cataloguing; they do not pass cohesion alone and are candidates for future sub-division (e.g. ai â†’ gpu-compute / agents / data; defi â†’ dex / lending / lsd / rwa / perps) once each sub-leaf reaches â‰Ą 3 live OKX members.
- `smart_contract_platform`, `information_technology`, and `defi` are mostly market beta; their statistical significance is driven by large n, not strong cohesion.
- This schema documents a **classification** (an input to a risk model's exposure matrix). It is not itself a covariance or factor risk model.

## Referential integrity

`python scripts/validate_schema.py` asserts at CI time:

- No duplicate `symbol` or `cg_id` in `sector.csv`.
- Every `sector` value is one of the 15 valid labels above.
- Every `role` value is `FACTOR` or `RESIDUAL`.
- `audit_from` present, parseable as date, non-null for all rows.
- `sector.parquet` column names and row count match `sector.csv` exactly.
- `sector_panel.parquet` column dtype is pandas `StringDtype` (not `object`).
- `sector_panel.parquet` column set equals `symbol` set in `sector.csv`.
- All symbols in `universe_tiers.json` are present in `sector.csv`.
- All symbols in `data/returns.parquet` are present in `sector.csv`.

## Reading files

```python
import pandas as pd

# Flat classification
sector = pd.read_csv("classification/sector.csv")
# or dtype-correct:
sector = pd.read_parquet("classification/sector.parquet")

# PIT panel â€” use this as group_neut input
panel = pd.read_parquet("classification/sector_panel.parquet")
# panel.dtypes â†’ all StringDtype; panel.loc["2025-01-15"].dropna() for a cross-section

# Returns
returns = pd.read_parquet("data/returns.parquet")

# Metadata
import json
roles = json.load(open("classification/sector_roles.json"))
tiers = json.load(open("classification/universe_tiers.json"))
```

Do **not** read `classification/sector_panel.parquet` with `pd.read_csv` â€” it does not exist as CSV. Do **not** call `.values` or `np.array()` on a `StringDtype` column expecting a numpy string array; use `.to_numpy(dtype=object, na_value=pd.NA)` or `.dropna().to_numpy()`.
